# Quick Task 260618-q81: Clean placeholder history + offer-duration price lines

## Objective

Two related changes to the price-history graph:

1. **Remove non-real (placeholder) data.** `data/price-history.jsonl` carries 3 seed rows
   authored in commit `6691055 "seed data files"` (2026-06-01 REWE, 2026-06-08 Kaufland,
   2026-06-15 REWE). They are synthesized placeholders, not scraped. Only the
   `2026-06-18 Kaufland` row is real (from the scheduled scrape, matches `current-offers.json`).
   Strip the 3 placeholders from `data/price-history.jsonl`, and sync the served dev copy
   `web/public/data/price-history.jsonl` (currently 3 stale placeholders, missing the real row)
   to the cleaned real history.

2. **Show offer DURATION as a price line over the valid week.** Today the chart plots one dot
   per offer at its scrape `date`. Change it so each offer renders as a **flat horizontal line
   at its price spanning `validFrom`→`validTo`** (Option A, user-chosen). Y = price, X = time,
   per-store colour kept. Distinct offers stay broken apart (no interpolation — HIST-02).

## Decisions (locked)

- **Chart form:** per-store horizontal price segment across each offer's valid week (uPlot kept;
  price axis kept). NOT Gantt bars (user changed mind from Gantt → line).
- **Break model:** the x-axis is built from every offer's `validFrom`/`validTo` (epoch seconds at
  UTC midnight) plus a `validTo + 1s` **sentinel** per offer. A store's series value at x is the
  offer price when x is inside one of that store's offer windows, else `null`. `spanGaps:false`
  then yields a flat segment per offer and a guaranteed break (the sentinel) between back-to-back
  same-store offers — so two separate offers are never joined by a misleading slope.
- **Markers-only-when-<3-points rule removed:** an offer is inherently a 2-point segment, so even
  a single offer shows a visible week-long line. Points stay shown so a one-day offer (from==to)
  is still visible as a dot.
- **Test independence:** `chart.test.mjs` currently reads the live `data/price-history.jsonl` and
  asserts the 3 seed rows. Rewrite it to use an inline multi-offer fixture (incl. a back-to-back
  same-store case to lock the sentinel break) so it is independent of the cleaned live data.
- Wasgau still never a series (HIST-03); cold-start "Noch keine Daten" panel unchanged.
