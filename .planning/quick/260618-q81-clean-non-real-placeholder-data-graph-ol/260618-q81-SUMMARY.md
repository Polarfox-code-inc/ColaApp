# Quick Task 260618-q81 — Summary

## What was done

**1. Removed placeholder (non-real) history data.** `data/price-history.jsonl` carried 3
synthesized seed rows from commit `6691055 "seed data files"` (06-01 REWE, 06-08 Kaufland,
06-15 REWE). Stripped them, leaving only the one genuinely scraped offer (06-18 Kaufland
10,99 €, valid 06-18..06-24 — matches `current-offers.json`). Synced the served dev copy
`web/public/data/price-history.jsonl`, which had the 3 placeholders and was missing the real row.
Relaxed the schema test's history guard from `>= 2 seeded` to `>= 1 real`.

**2. Offer duration as a price line (user-chosen Option A).** Reworked the chart so each offer
renders as a flat price segment spanning `validFrom`→`validTo` instead of a single dot at the
scrape date. `prepareChartData` now builds the x-axis from every offer's window boundaries plus a
one-second sentinel after each `validTo`; a store's series carries the price inside its windows and
`null` elsewhere. With `spanGaps:false` that paints a horizontal line over the whole valid week and
breaks between distinct offers — preserving the no-interpolation honesty rule (HIST-02). The
sentinel guarantees two back-to-back same-store offers never join into one sloped line. Dropped the
old "<3 points → markers only" rule (an offer is inherently a segment); points stay shown so a
one-day offer is still visible. New return shape `{data, x, stores, offerCounts, isEmpty}`.
