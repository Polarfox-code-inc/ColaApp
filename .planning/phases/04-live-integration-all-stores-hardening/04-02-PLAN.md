<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub-hosted runner → public repo (git push) | CI commits scraped third-party data back to a public repo via GITHUB_TOKEN |
| marktguru.de → CI runner (headless fetch) | Untrusted external JSON crosses into CI; keys scraped at runtime |
| CI build artifact → public GitHub Pages CDN | The deployed shell + data become world-readable |
| third-party GitHub Actions → CI runner | Supply-chain: action code executes with the workflow token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Elevation of privilege | over-broad GITHUB_TOKEN | mitigate | Top-level `permissions: contents: write` only; deploy job narrows to `pages: write` + `id-token: write` + `contents: read` (least privilege) |
| T-04-06 | Denial of service | recursive workflow trigger / runaway minutes | mitigate | GITHUB_TOKEN-pushed commits do not re-trigger workflows (verified) + `[skip ci]` + `concurrency` serialization |
| T-04-07 | Tampering / Elevation | compromised or typo-squatted third-party action | mitigate | Only first-party `actions/*`, pinned to verified majors; no third-party keepalive action (use built-in git/gh) |
| T-04-08 | Information disclosure | marktguru keys / secrets leaked in CI logs | mitigate | No new secrets; scraper logs only err.message and never the scraped x-apikey/x-clientkey (existing T-02-10 guard); headless fetch, sane User-Agent (L-6) |
| T-04-09 | Tampering / Integrity | corrupt or half-written data/ committed | mitigate | Scraper's atomic temp+rename + validate-before-write preserved; CI `git add`s whole files only; concurrency + `git pull --rebase` prevent push races |
| T-04-SC | Tampering | npm/GitHub Action installs | mitigate | `npm ci` lockfile-exact; zero new npm packages; all actions first-party per RESEARCH Package Legitimacy Audit (no [ASSUMED]/[SUS]/[SLOP]) |
</threat_model>
