<objective>
Build the I/O boundary of the scraper: the network module (`fetch.mjs`) that scrapes the marktguru bootstrap keys, makes the single offers/search call, and rides out transient failures with a per-attempt-timeout exponential-backoff retry; and the filesystem module (`io.mjs`) that reads the prior data snapshot, writes whole files atomically via temp+rename, and appends JSONL history lines. These are the only two impure-ish modules besides the orchestrator; everything else (Plan 01) is pure.

Purpose: Isolates all network and disk side effects behind two injectable seams so the orchestrator (Plan 03) and its tests can run entirely offline against the captured fixture. Encodes the D-11 retry policy, D-12 key hygiene, and the atomic-write + append-only-history resilience rules (DATA-05).
Output: `scraper/fetch.mjs`, `scraper/io.mjs`, and `test/scraper.fetch.test.mjs`.
</objective>
