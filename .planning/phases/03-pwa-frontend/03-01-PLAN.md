## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → web/ build | Third-party packages (vite, vite-plugin-pwa, uplot) enter the build |
| installed SW → cached responses | Service worker mediates what the installed app serves offline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Stale-forever shell/data (PWA-03 regression) | mitigate | registerType:'autoUpdate' for shell + StaleWhileRevalidate (never CacheFirst) for /data/* so fresh data revalidates in background (RESEARCH Pitfall 5) |
| T-03-02 | Info disclosure | Over-broad SW scope | mitigate | scope:'./', caching-only SW, no Notification/Push permissions (D-22) |
| T-03-SC | Tampering | npm installs (vite/vite-plugin-pwa/uplot) | mitigate | Pin exact major versions per RESEARCH Package Legitimacy Audit (all OK; vite SUS is a recency false-positive → pin ^7); no package has a postinstall script (verified). vite SUS is auto-approvable per researcher note (no [ASSUMED]/[SUS] requiring a blocking gate). |

All other STRIDE categories (Spoofing, Repudiation, Elevation, and most of Info-disclosure/DoS) are N/A: no auth, no sessions, no backend, no secrets in the client (marktguru keys live only in the scraper).
</threat_model>
