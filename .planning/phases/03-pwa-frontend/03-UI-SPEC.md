# Phase 3 — UI Design Contract

> Visual and interaction contract for the ColaApp PWA frontend. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Scope: a single-screen, single-user German PWA for one Android phone. Built with Vite + `vite-plugin-pwa` + **vanilla JS** (no framework, no component library — locked by CONTEXT D-01) against the six Phase 1 mocks. **Neutral / minimal utility look — NOT Coca-Cola branded.**

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla JS + hand-authored CSS; no shadcn/Tailwind — CONTEXT D-01, CLAUDE.md "What NOT to Use") |
| Preset | not applicable |
| Component library | none (framework-free; DOM built directly — CONTEXT D-01 + Claude's-discretion decomposition) |
| Icon library | none as a dependency. Use a tiny inline-SVG set authored in `web/` (status icons + legend swatches). Inline SVG keeps the PWA shell small and offline-cacheable. |
| Font | System font stack — no web-font download (offline-first, zero blocking request): `font-family: system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif`. Tabular figures for prices: `font-variant-numeric: tabular-nums`. |

**shadcn gate note:** stack is Vite, which normally triggers the shadcn initialization gate. Gate is **resolved as `Tool: none`** because CONTEXT.md D-01 explicitly locks "neutral/minimal, vanilla JS, no component library." Re-asking would violate the locked-decision rule. Registry safety gate: not applicable.

**CSS approach (Claude's discretion, CONTEXT):** plain CSS with custom properties (CSS variables) for all tokens below, declared once on `:root`. No preprocessor, no CSS framework. A single `web/src/styles.css` (or small per-component CSS modules imported by Vite) is sufficient for one screen.

## Spacing Scale

Declared values (multiples of 4; 8-point base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge inner padding, chip gaps |
| sm | 8px | Compact spacing inside a card row (price ↔ €/l) |
| md | 16px | Default: card inner padding, gap between stacked fields, body rhythm |
| lg | 24px | Gap between store cards, hero inner padding |
| xl | 32px | Gaps between the four page sections (hero ↔ cards ↔ graph ↔ footer) |
| 2xl | 48px | Top/bottom page padding on the single scroll column |
| 3xl | 64px | Reserved (empty-state vertical breathing room, e.g. cold-start graph) |

**Exceptions:**
- **Touch targets:** any tappable element (none required by CONTEXT — the app is read-only with no CTAs — but if a future affordance like a legend toggle appears it) must be ≥ **44×44px** hit area (`min-height: 44px`), independent of visual size. This is the one allowed non-8pt value.
- **Page max content width:** `max-width: 480px`, centered (`margin-inline: auto`). The app is phone-first; on a wider viewport it stays a single narrow column rather than stretching.
- **Border radius scale (not spacing, declared here for completeness):** `--radius-sm: 8px` (badges/chips), `--radius-md: 12px` (cards, hero). Single 1px hairline borders use `--color-border`.

## Typography

Exactly **4 sizes, 2 weights** (regular 400 + semibold 600). Body line-height 1.5, heading/numeric line-height ~1.2.

| Role | Size | Weight | Line Height | Used for |
|------|------|--------|-------------|----------|
| Display | 28px | 600 | 1.2 | The hero best-deal **price** (`€9,99`) — the single biggest number on screen |
| Heading | 20px | 600 | 1.25 | Section labels ("Bestes Angebot", "Preisverlauf"), store name on a card |
| Body | 16px | 400 | 1.5 | Default text: valid-until line, €/l, hero supporting line, footer text |
| Label | 13px | 600 (state labels) / 400 (meta) | 1.3 | State chips ("kein Angebot", "Fehler"), "demnächst ab …" badge, axis/legend text, freshness meta |

Notes:
- No size below 13px anywhere (mobile legibility floor). uPlot axis tick labels and the legend use the 13px Label size.
- Prices and €/l always render with `tabular-nums` so digits align across the 5 cards.
- Only two weights total (400, 600). Do not introduce 500/700.

## Color

Single neutral light theme (no in-app dark mode — UI-01 deferred to v2). 60/30/10 split. The **accent (10%) is a calm green reserved exclusively for the "best/active deal" affirmative signal** — chosen over Coca-Cola red so the UI stays un-branded (CONTEXT D-01); red is not used in the body at all. The three store states are carried by **three separate semantic colors** (info / warning / muted), each paired with an icon + text label so state never relies on color alone (WCAG 1.4.1).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Page background, hero surface, graph plot area |
| Secondary (30%) | `#F4F5F7` | Store card surfaces, footer surface (the repeated 5-card field) |
| Accent (10%) | `#0B7A3B` (green) | See reserved-for list below |
| Destructive | not used | No destructive actions exist in this phase (read-only app). Field left intentionally empty. |

**Accent (`#0B7A3B`) reserved for — explicit list (never "all interactive elements"):**
- The hero "best deal" price + its left accent bar/marker when an active offer exists.
- The active-offer card's price value and its thin left state-edge.
- The "✓ aktiv" / active-offer chip on a store card.
- The lowest line/legend swatch is NOT forced green — per-store lines keep their own palette (below); accent green is for the *active-deal affirmation*, not for the graph.

**Semantic state tokens (store cards + freshness; each always paired with icon + label):**

| Semantic | Text/Icon | Surface tint | Meaning | WCAG-AA (normal text) |
|----------|-----------|--------------|---------|-----------------------|
| Active / best deal | `#0B7A3B` on `#FFFFFF` | — | `status: offer`, currently valid | 5.44 ✓ |
| Info (unavailable) | `#1B5FB0` on `#E8F0FB` | `#E8F0FB` | `unavailable` = Wasgau, "nicht automatisch verfügbar" — expected, an "i" icon, NOT an error | 5.52 ✓ |
| Warning (error) | `#9A5B00` on `#FCF1DC` | `#FCF1DC` | `error` = fetch failed, amber "!" | 4.85 ✓ |
| Muted (no offer) | `#595E66` on `#ECEDEF` | `#ECEDEF` | `no_offer` = "kein Angebot", grey, dash icon | 5.57 ✓ |
| Upcoming badge | `#1B5FB0` on `#E8F0FB` | `#E8F0FB` | "demnächst — ab Mo 23.06." (derived future `validFrom`) | 5.52 ✓ |
| Stale marker | `#9A5B00` on `#FCF1DC` | `#FCF1DC` | per-store `lastUpdated` older than 3 days; small "⚠ veraltet" chip on that card only | 4.85 ✓ |

**Neutral text + structure tokens:**

| Token | Value | WCAG-AA | Usage |
|-------|-------|---------|-------|
| `--color-text` | `#1A1D21` | 16.91 on white ✓ | Primary body/heading text |
| `--color-text-muted` | `#5B616B` | 6.24 on white / 5.72 on card ✓ | Secondary meta (valid dates, €/l label, footer) |
| `--color-border` | `#D8DADE` | hairline (not text) | 1px card/section dividers |
| `--color-theme-dark` | `#1A1D21` | 16.91 with white ✓ | **PWA chrome only** (manifest `theme_color`, status-bar tint, splash) — CONTEXT D-20. NOT a UI background. |

**uPlot per-store line palette (stable mapping — Claude's discretion, CONTEXT D-12). Wasgau has NO line.** Chosen for legibility on a white plot on a small screen and to be distinguishable for the most common red/green color-vision deficiency (each pair also differs in lightness; markers double-encode by shape):

| Store | Line color | Marker shape | Contrast on white |
|-------|-----------|--------------|-------------------|
| REWE | `#C2143C` (red) | circle | high |
| Edeka | `#1B5FB0` (blue) | square | high |
| Lidl | `#0B7A3B` (green) | triangle | high |
| Kaufland | `#7A3FB0` (violet) | diamond | high |
| Wasgau | — (no line; appears in legend greyed as "nicht automatisch verfügbar") | — | — |

> Legend must show all 5 stores; Wasgau is rendered greyed/struck with the "nicht automatisch verfügbar" note so its absence is explained, not silently missing.

## Copywriting Contract

All copy is **hardcoded German** (CONTEXT D-03), `de-DE` number/date formatting via `Intl` (D-04). There is **no primary CTA** — the app is a read-only glance tool with no buttons/forms. The "CTA" row below is therefore the one persistent affirmative the app exists to deliver.

| Element | Copy |
|---------|------|
| Primary CTA | *(none — read-only app).* The equivalent affirmative is the hero best-deal line: **`{Store} — {€Preis} — gültig bis {Wochentag TT.MM.} — {€/l}`** (e.g. `REWE — €9,99 — gültig bis Sa 21.06. — 0,83 €/l`) |
| Empty state heading | Hero, no active offer: **`Kein aktuelles Angebot`** |
| Empty state body | If an upcoming offer exists, nudge: **`Nächstes Angebot ab {Wochentag TT.MM.} bei {Store}.`** If none: **`Zurzeit ist der 12×1-l-Kasten nirgends im Angebot.`** |
| Error state | Store card, `error`: chip **`Fehler`** + line **`Konnte nicht abgerufen werden.`** (Carried-forward last-known data, if any, may still show beneath with its own stale marker.) |
| Destructive confirmation | not applicable — no destructive actions in this phase |

**Full German string table (the executor uses these verbatim):**

| Key | German string | Where |
|-----|---------------|-------|
| `hero.title` | `Bestes Angebot` | Hero section label |
| `hero.empty` | `Kein aktuelles Angebot` | Empty hero heading |
| `hero.nudge` | `Nächstes Angebot ab {Wochentag TT.MM.} bei {Store}.` | Empty hero, upcoming exists |
| `hero.none` | `Zurzeit ist der 12×1-l-Kasten nirgends im Angebot.` | Empty hero, no upcoming |
| `hero.validUntil` | `gültig bis {Wochentag TT.MM.}` | Hero + active card |
| `card.perLitre` | `{0,83} €/l` | Active card + hero |
| `state.no_offer` | `kein Angebot` | Card chip (muted) |
| `state.unavailable` | `nicht automatisch verfügbar` | Card chip (info "i") |
| `state.error` | `Fehler` | Card chip (warning "!") |
| `state.active` | `aktiv` | Active-offer chip (accent ✓) |
| `badge.upcoming` | `demnächst — ab {Wochentag TT.MM.}` | Upcoming badge on card |
| `stale.marker` | `veraltet` | Per-store stale chip (⚠) |
| `graph.title` | `Preisverlauf` | Graph section label |
| `graph.empty` | `Noch keine Daten` | Cold-start graph heading |
| `graph.emptyBody` | `Sobald Preise gesammelt sind, erscheint hier der Verlauf.` | Cold-start graph body |
| `graph.legendUnavailable` | `nicht automatisch verfügbar` | Wasgau legend note |
| `footer.updated` | `zuletzt aktualisiert: {TT.MM.JJJJ HH:MM} Uhr` | Freshness footer (file-level `lastUpdated`) |
| `unit.case` | `12×1-l-Kasten` | Product reference where named |

**Formatting rules (CONTEXT D-04, locked):**
- Price: integer cents ÷ 100 → `Intl.NumberFormat('de-DE',{style:'currency',currency:'EUR'})` → `9,99 €` style. (Display order in hero/copy uses `€9,99` per D-05 example — match D-05.)
- €/litre: `pricePerLitre` cents ÷ 100 → `0,83 €/l` (comma decimal, space, `€/l`).
- Date: `YYYY-MM-DD` → `TT.MM.JJJJ` (e.g. `16.06.2026`); short weekday form `Mo 21.06.` via `Intl.DateTimeFormat('de-DE',{weekday:'short'})` + `TT.MM.`.
- Timestamp: ISO-UTC `lastUpdated` → absolute `de-DE` `15.06.2026 06:00 Uhr` (CONTEXT D-17; no relative "vor 2 Stunden").

## Component Anatomy & Interaction Contract

> The executor builds these as plain DOM modules (hero / card / graph / freshness — Claude's-discretion decomposition). Order is fixed top→bottom (CONTEXT D-02): **hero → 5 store cards → price-history graph → freshness footer.**

### Page shell
- Single vertical scroll column, `max-width: 480px`, centered, `padding: 48px(2xl) 16px(md)`.
- Section gap = `32px (xl)`.
- White (`#FFFFFF`) page background.

### Hero (OFFR-01 — D-05/06/07)
- Section label `Bestes Angebot` (Heading 20/600).
- **Active state** (≥1 `status:offer` with `validFrom ≤ today ≤ validTo`, `needsReview:false`, lowest price wins):
  - Big price (Display 28/600, accent green) + store name (Heading 20/600).
  - Supporting line (Body 16/400, muted): `gültig bis {Wochentag TT.MM.} · {€/l}`.
  - Thin 3px accent-green left edge on the hero card.
- **Empty state** (no active offer — the common real case): `Kein aktuelles Angebot` (Heading) + nudge/none body (Body, muted). No green. Neutral surface.
- `needsReview:true` offers are filtered out before hero selection (CONTEXT D-07, Phase 1 D-08).

### Store card ×5 (always all five — OFFR-02/03/04, D-08/09/10/13)
Anatomy (each card, `#F4F5F7` surface, `--radius-md`, `16px` padding, 1px left state-edge in the state color):
1. **Row 1:** store `displayName` (Heading 20/600) + state chip (right-aligned): icon + label in the state's semantic color, on its tint, `--radius-sm`.
2. **Row 2 (active only):** price (Heading 20/600, accent green) + `€/l` (Body, muted) + `gültig bis {…}` (Body, muted).
3. **Upcoming badge (D-13):** if derived `upcoming` (future `validFrom`), an info badge `demnächst — ab {Wochentag TT.MM.}` replaces the active price row; card still sorts above `no_offer`.
4. **Stale marker (D-18):** if this store's `lastUpdated` > 3 days old, a small `⚠ veraltet` warning chip appears in Row 1 next to the state chip. **Per-store only — no global banner** (OFFR-07 deferred).

**State chips (icon + color + label — must be unmistakable, OFFR-03/04):**
| status | icon | label | color/tint |
|--------|------|-------|-----------|
| offer (active) | ✓ | `aktiv` | accent green |
| upcoming (derived) | → / clock | `demnächst — ab {…}` | info blue / `#E8F0FB` |
| no_offer | – (dash) | `kein Angebot` | muted `#595E66` / `#ECEDEF` |
| unavailable | i | `nicht automatisch verfügbar` | info blue / `#E8F0FB` |
| error | ! | `Fehler` | warning amber / `#FCF1DC` |

**Card ordering (D-09):** active (cheapest-first) → upcoming → no_offer → unavailable/error. Recompute on render.

### Price-history graph (HIST-01/02/03 — D-11/12/13-graph/14)
- Section label `Preisverlauf` (Heading).
- **uPlot** canvas chart, full content width, fixed height (~`220–260px`), white plot area. Time x-axis, price (€) y-axis. Sparse de-DE axis labels (few ticks; 13px Label).
- **Per-store lines** (REWE/Edeka/Lidl/Kaufland) using the stable palette + marker shapes above. **Wasgau never gets a line.**
- **Sparse/gap rules (D-13-graph):** 1–2 points for a store → markers only (no connecting line); ≥3 points → connect; a no-offer stretch → **break the line** (uPlot `null`/`NaN` gap), never interpolate across it.
- **Window:** all available history (D-14). No pan/zoom.
- **Legend:** below the chart, all 5 stores with swatch + name; Wasgau greyed with `nicht automatisch verfügbar`.
- **Cold-start (0 points — Claude's discretion):** render `Noch keine Daten` heading + `Sobald Preise gesammelt sind, erscheint hier der Verlauf.` body in the chart area — **do not** draw an empty axis.

### Freshness footer (OFFR-06 — D-16/17/18)
- File-level `lastUpdated` (CurrentOffers root / status file) → `zuletzt aktualisiert: {TT.MM.JJJJ HH:MM} Uhr` (Body, muted), centered.
- This is the "job is alive" timestamp (Phase 2 D-05, file-level always bumps). Per-store staleness is shown on the cards, not here.

### Responsive behavior (phone-first)
- One column at all widths; `max-width: 480px` cap. No breakpoints needed for layout.
- Cards stack vertically full-width.
- uPlot must resize to container width on load and `resize` (set width from `clientWidth`).
- Respect safe-area insets where present (`padding` + `env(safe-area-inset-*)`) for standalone display.

### PWA chrome (PWA-01 — D-19/20/21)
- App name `ColaApp`; `short_name` `ColaApp`.
- `display: standalone`, `start_url: "./"` (or repo subpath in Phase 4).
- `theme_color: #1A1D21` (neutral dark — D-20), `background_color: #FFFFFF`.
- Icons: 192 + 512 + maskable. **Artwork (Claude's discretion):** a simple, trademark-safe glyph — a generic single-bottle silhouette (chosen over the case for clarity at 192px) on a solid `#1A1D21` tile, light glyph — **NOT** the real Coca-Cola logo/wordmark/red.
- Service worker (`vite-plugin-pwa` `generateSW`, Workbox 7): precache shell; **StaleWhileRevalidate** runtime route for `data/*.json(l)` (PWA-02 offline last-data; PWA-03 fresh-when-online). Caching only — **no push/notifications** permissions (D-22).

## Accessibility Notes (carried for checker + executor)

- State is **never color-only**: every store state and the upcoming/stale signals pair color with an icon and a German text label (WCAG 1.4.1).
- All text/icon-on-tint pairs verified WCAG-AA for normal text (ratios in the Color table; lowest is 4.85 amber-on-amber-tint).
- Minimum text size 13px; primary content 16px; tabular numerals for price alignment.
- uPlot canvas is decorative-adjacent; the same best-deal facts are available as text in the hero and cards (graph is not the sole carrier of any required information — HIST is supplementary to OFFR).
- No motion/animation required; if any is added, keep it subtle and honor `prefers-reduced-motion`.
