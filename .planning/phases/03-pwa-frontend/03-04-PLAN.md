<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fetched JSON/JSONL → DOM | Untrusted-treated data rendered into the page |
| network/cache → app | fetch may fail or return malformed/cached data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering (DOM XSS) | Rendering data text as HTML | mitigate | All render modules use textContent/createElement; no innerHTML with data values (ASVS V5); store names come from the frozen STORES allow-list |
| T-03-08 | DoS (availability) | A failed/malformed fetch blanks the whole screen | mitigate | loadData degrades per-file; main.js wraps the flow to render per-store error/cold-start states, never a white screen (ASVS V7) |
| T-03-09 | Info disclosure | marktguru keys leaking into the client | accept→N/A | The PWA never calls marktguru and embeds no keys; it reads only static JSON (keys live solely in the scraper) |

Spoofing/Repudiation/Elevation are N/A (no auth, no sessions, no privileged operations). Defensive contract validation (Task 1) is the integrity control on the data shape.
</threat_model>
